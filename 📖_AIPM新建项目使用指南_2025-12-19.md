## ✨ 新建项目详细步骤

### 第一步：打开新建对话框

**方法1**：点击页面右上角的 **[新建项目]** 按钮

**对话框内容**：
- 标题：新建项目
- 宽度：700px
- 包含：基本信息 + WBS任务分解

---

### 第二步：填写基本信息

#### 1. 项目名称 ⭐必填
```
示例：明升智能拧紧系统V3.0
      某汽车厂总装线改造项目
      航空航天拧紧解决方案
```

#### 2. 项目经理 ⭐必填
```
示例：张经理、李总监、王主管
```

#### 3. 开始日期 ⭐必填
- 点击日期选择器
- 选择项目计划开始日期
- 格式：YYYY-MM-DD

#### 4. 计划完成日期 ⭐必填
- 点击日期选择器
- 选择项目计划结束日期
- 必须晚于开始日期

#### 5. 项目预算
```
示例：500万元
输入数字即可，自动显示"万元"单位
```

#### 6. 团队人数
```
默认：5人
可调整：使用数字输入框
最小值：1人
```

#### 7. 关键路径
```
示例：需求分析 -> 系统设计 -> 开发实施 -> 测试验证 -> 上线交付
      现场勘察 -> 方案设计 -> 设备采购 -> 安装调试 -> 验收
```

#### 8. 项目描述
```
简要说明项目背景、目标、范围
支持多行文本输入(3行高度)
```

---

### 第三步：WBS任务分解

#### 什么是WBS？
WBS (Work Breakdown Structure) = 工作分解结构
将项目分解为可管理的任务单元

#### 操作步骤

1. **点击"添加任务"按钮**
   - 表格中出现新的一行

2. **填写任务信息**
   | 字段 | 说明 | 示例 |
   |------|------|------|
   | 任务名称 | 具体工作内容 | 需求分析、系统设计、前端开发 |
   | 状态 | 当前状态 | 未开始/进行中/已完成 |
   | 进度(%) | 完成百分比 | 0-100 |

3. **添加多个任务**
   - 建议3-10个任务
   - 按工作顺序添加
   - 涵盖项目主要阶段

4. **删除任务**
   - 点击任务右侧的"删除"按钮

#### 示例WBS

```
【软件项目】
1. 需求分析      - 未开始 - 0%
2. 系统设计      - 未开始 - 0%
3. 前端开发      - 未开始 - 0%
4. 后端开发      - 未开始 - 0%
5. 测试验证      - 未开始 - 0%
6. 部署上线      - 未开始 - 0%

【工程项目】
1. 现场勘察      - 未开始 - 0%
2. 方案设计      - 未开始 - 0%
3. 设备采购      - 未开始 - 0%
4. 现场施工      - 未开始 - 0%
5. 调试验收      - 未开始 - 0%
```

---

### 第四步：提交创建

1. **点击"确定创建"按钮**

2. **系统自动验证**
   - ✅ 项目名称不能为空
   - ✅ 项目经理不能为空
   - ✅ 必须选择开始和结束日期
   - ❌ 未通过显示警告提示

3. **创建成功**
   - 提示：项目创建成功！
   - 对话框自动关闭
   - 项目出现在列表顶部
   - 甘特图自动刷新

4. **系统自动设置**
   ```javascript

   状态: 进行中
   已投入: 0万元
   AI预测准确度: 85%
   风险等级: 低
   ```

## 📊 甘特图使用指南

### 视图模式切换

**位置**：项目甘特图卡片顶部

**三种模式**：

1. **日视图**
   - 时间轴：月/日 (例如 12/19)
   - 适用场景：短期项目(1-3个月)
   - 精度：天级别

2. **周视图** ⭐推荐
   - 时间轴：月第N周 (例如 12月3周)
   - 适用场景：中期项目(3-12个月)
   - 精度：周级别

3. **月视图**
   - 时间轴：年-月 (例如 2025-01)
   - 适用场景：长期项目(1年以上)
   - 精度：月级别

### 甘特图元素解读

#### 1. 项目条
```
┌─────────────────────────────────────┐
│ ████████████░░░░░░░ 65%             │ ← 项目名称
└─────────────────────────────────────┘
  ▲           ▲       ▲
  │           │       │
开始时间   当前进度  结束时间

深色部分 = 已完成进度
浅色部分 = 未完成部分
中央数字 = 完成百分比
```

#### 2. 颜色含义

| 颜色 | 状态 | 含义 |
|------|------|------|
| 🔵 蓝色 (#409EFF) | 进行中 | 项目正常推进 |
| 🟢 绿色 (#67C23A) | 已完成 | 项目成功交付 |
| 🔴 红色 (#F56C6C) | 延期风险 | 进度落后，需关注 |

#### 3. 当前时间线

```
                         ┆ ← 红色虚线
                      今天
┌──────────────────────┆────────────┐
│ ████████████████░░░░░┆░░░░░░░░░░░ │
└──────────────────────┆────────────┘
                       ▲
               用于判断是否延期
```

#### 4. Tooltip详情

**触发**：鼠标悬停在项目条上

**显示内容**：
```
━━━━━━━━━━━━━━━━━━━━
  明升智能拧紧系统V2.0
━━━━━━━━━━━━━━━━━━━━
  项目经理: 张经理
  状态: 进行中
  计划进度: 65%
  实际进度: 72.3%
  开始日期: 2024-10-01
  结束日期: 2025-01-31
  总工期: 122天
━━━━━━━━━━━━━━━━━━━━
```

**实际进度计算**：
```
实际进度 = (今天 - 开始日期) / (结束日期 - 开始日期) × 100%
```

## 📈 项目健康度分析

### 雷达图说明

**位置**：向下滚动到"AI项目管家分析"卡片

**5个维度评估**：

1. **进度达成** (Progress)
   - 数值来源：项目实际进度
   - 满分：100分

2. **质量评分** (Quality)
   - 数值来源：AI评估(85-95分)
   - 评估因素：代码质量、测试覆盖率等

3. **成本控制** (Cost)
   - 计算公式：(1 - 已投入/预算) × 100
   - 满分：预算使用率0%
   - 0分：预算超支

4. **团队协作** (Teamwork)
   - 数值来源：AI评估(90-100分)
   - 评估因素：沟通效率、任务分配等

5. **风险管理** (Risk)
   - 低风险：95分
   - 中风险：75分
   - 高风险：60分

### 如何解读

**理想状态**：
```
    进度达成
       100
        ╱│╲
       ╱ │ ╲
      ╱  │  ╲
  团队 ─ ● ─ 质量
     ╲  │  ╱
      ╲ │ ╱
       ╲│╱
    成本─风险
```
所有维度接近满分，形成正五边形

**预警状态**：
```
某个维度明显低于其他维度
→ 需要重点关注和改进
```

## ⚠️ 常见问题

### Q1：创建后如何修改项目？
**A**：当前版本支持查看和创建。后续版本将支持编辑功能。

### Q2：甘特图不显示？
**A**：
1. 检查是否切换到"列表视图"
2. 确认至少有1个项目
3. 刷新页面或切换视图模式

### Q3：WBS任务可以有层级吗？
**A**：当前版本为平铺结构。展开项目列表可查看WBS任务树形结构。

### Q4：项目进度如何更新？
**A**：点击项目列表的"更新"按钮(后续版本完善)。

### Q5：甘特图时间轴太密集？
**A**：切换到"周视图"或"月视图"以获得更好的显示效果。

## 💡 最佳实践

### 1. 项目命名规范
```
✅ 推荐：明升AICRM V4.0升级
✅ 推荐：某汽车厂焊装线改造-2025Q1
❌ 避免：项目1、新项目
```

### 2. WBS分解原则
- **适度细化**：不要太粗(少于3个)或太细(超过20个)
- **逻辑清晰**：按时间顺序或功能模块分解
- **可交付**：每个任务有明确的输出成果

### 3. 日期选择建议
- **留出缓冲**：预留10-20%的时间余量
- **避开节假日**：考虑法定节假日影响
- **分阶段**：大项目可分为Phase1、Phase2等

### 4. 预算管理
- **预留应急**：预算预留15-20%应急资金
- **分项预算**：大项目建议细化到各个阶段
