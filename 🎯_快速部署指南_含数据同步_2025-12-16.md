## 🔧 配置完整演示数据（可选）

### 步骤1：准备数据

1. **启动本地服务器**
   ```bash
   npm run dev
   ```

2. **配置所有内容**
   - 访问：http://localhost:3002/admin
   - 登录后台（默认可直接进入）
   - 添加所有演示内容：
     - ✅ 产品信息
     - ✅ 服务介绍
     - ✅ 课程内容
     - ✅ Banner图片
     - ✅ 明星产品
     - ✅ 等等...

### 步骤2：导出数据

1. **访问数据管理工具**
   ```
   http://localhost:3002/import-demo-data.html
   ```

2. **导出数据**
   - 点击 "📥 导出当前数据"
   - 或点击 "💾 下载为文件"
   - 复制所有导出的代码

### 步骤3：更新演示数据

1. **打开文件**
   ```
   src/utils/initDemoData.js
   ```

2. **找到 `demoData` 对象**（约第11行）

3. **替换为您导出的数据**
   ```javascript
   const demoData = {
     demoDataVersion: '1.0.1', // 递增版本号
     
     // 粘贴导出的数据（注意保持JSON格式）
     cms: '{"homeBanners":[...]}',
     products: '{"list":[...]}',
     classroom: '{"courses":[...]}',
     // ... 其他数据
   }
   ```

4. **保存文件**

### 步骤4：重新构建和部署

1. **构建**
   ```bash
   npm run build
   ```

2. **重新部署**
   - 再次访问：https://app.netlify.com/drop
   - 拖拽新的 `dist` 文件夹
   - 或运行：`一键部署到Netlify.bat`

3. **验证**
   - 访问部署后的网站
   - 刷新页面（Ctrl+F5 强制刷新）
   - 确认所有数据都正确显示

## 📱 功能说明

### ✅ 所有访客可以做的事

无论是否配置演示数据，访客都可以：

1. ✅ **浏览所有页面** - 无时间限制
2. ✅ **点击所有功能** - 完全正常使用
3. ✅ **访问后台** - `/admin` 路径
4. ✅ **修改数据** - 仅在他们的浏览器中生效
5. ✅ **长时间操作** - 没有任何限制

### ⚠️ 数据独立性

- 每个访客的浏览器有独立的数据存储
- A用户的操作不会影响B用户
- 管理员的配置不会自动同步到访客

### ✅ 演示数据的作用

- 首次访问时自动加载预设的内容
- 确保所有人看到相同的初始状态
- 访客可以在此基础上进行操作（仅本地生效）

## 🆘 常见问题

### Q：部署后访客看不到我添加的产品？
**A：** 因为数据保存在您的浏览器中。请按"配置完整演示数据"步骤导出并集成数据。

### Q：访客可以长时间使用网站吗？
**A：** 可以！没有任何时间限制，所有功能都可以正常使用。

### Q：访客的操作会影响其他人吗？
**A：** 不会。每个访客的数据是独立的。

### Q：如何更新网站内容？
**A：** 修改数据后，重新导出→更新`initDemoData.js`→构建→部署。

### Q：可以隐藏后台入口吗？
**A：** 可以，但建议使用真实的用户认证系统（需要后端支持）。
