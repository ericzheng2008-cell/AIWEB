设计一个 单个销售漏斗的优化结构，使其 自上而下呈现漏斗阶段，每一阶段都包含 赢率分析、标志事件以及客户主动性与个人画像分析。以下是详细方案，可直接用于 Figma 原型设计和系统落地。

1. 单个销售漏斗优化结构（Figma / UI 设计）
1.1 顶层布局（从上到下）
Single Sales Funnel Frame（宽度可约 600px）
  ├── Header（商机基本信息 + 总赢率）
  ├── Funnel Stage 1: 潜在客户
  │       ├── Stage WinRate
  │       ├── Key Events（标志事件列表，可人工输入）
  │       ├── Customer Activity Analysis（主动性评分）
  │       └── Customer Persona Insights（兴趣、喜好、背景）
  ├── Funnel Stage 2: 联系人
  │       ├── Stage WinRate
  │       ├── Key Events
  │       ├── Customer Activity Analysis
  │       └── Customer Persona Insights
  ├── Funnel Stage 3: 首次接触
  │       └── 同上
  ├── Funnel Stage 4: 技术交流
  │       └── 同上
  ├── Funnel Stage 5: 再次预约/拜访
  │       └── 同上
  ├── Funnel Stage 6: 决策/签单
  │       └── 同上
  └── Footer: AI Recommended Actions
1.2 每一阶段模块设计
A. Stage Header
Stage Name（阶段名称，如“首次接触”）
Stage WinRate（例如 68%，趋势箭头 +变化百分比）
Stage Status Tag（高/中/低风险）
B. Key Events（标志事件）
支持人工输入事件 + 系统自动事件
示例：
预约情况
拜访情况
技术交流
再次预约拜访
每个事件包含：
时间戳
事件类型（预约/拜访/交流等）
对赢率影响（+/-百分比）
备注（可手动填写）
C. Customer Activity Analysis（客户主动性分析）
自动计算客户主动性评分：
邮件回复率
拜访响应速度
会议参与度
主动提出需求/问题
可视化：
小型雷达图 / 条形图展示主动性维度
越活跃 → 得分越高 → 对赢率正向加权
D. Customer Persona Insights（客户个人画像）
展示客户背景和偏好：
角色/职位
决策权级别
兴趣偏好（例如偏技术方案、价格敏感度）
风格偏好（沟通方式、偏好频率等）
AI 分析：
对客户态度评分（积极、中立、消极）
预测下一步可能行为（如是否会主动预约）
1.3 人机交互设计
人工输入标志事件：
每阶段可点击“添加事件” → 弹出表单：
事件类型、时间、备注、影响赢率
事件与赢率联动：
新增事件 → 自动重新计算该阶段 WinRate
客户画像更新：
AI 根据新增事件更新态度、主动性和偏好分析
阶段间预测：
系统根据当前阶段赢率和事件趋势 → 预测下一阶段赢率
AI 行动建议：
根据每阶段分析 → 给出下一步操作提示
例如：“安排二次技术交流会议”、“发定制方案邮件”
1.4 漏斗可视化建议
纵向漏斗，顶部潜在客户 → 底部签单
每阶段：
左侧显示阶段 WinRate + 趋势
中间为漏斗形状图示
右侧显示 Key Events + 客户分析面板
颜色提示：
阶段赢率高：绿色
中等：黄色
低：红色
互动功能：
点击事件 → 展开详细备注
hover 客户画像 → 展示个人偏好、兴趣、历史行为
1.5 AI 计算逻辑建议
阶段赢率计算公式：
StageWinRate = BaseStageRate 
               + Σ(EventImpact)
               + CustomerActivityScore * 影响权重
               + PersonaMatchScore * 影响权重
客户主动性评分：
0~100 分，越高表示越积极
影响阶段赢率
态度分析：
积极/中立/消极 → 影响权重可调
可结合人工标注和 AI 推理
跨阶段预测：
利用当前阶段 WinRate + 客户主动性 + 历史类似商机数据 → 预测下一阶段成功概率

自上而下清晰呈现阶段
每阶段均衡融合人工事件输入与 AI 分析
实时计算赢率并提示风险
客户主动性和个人画像直接影响分析和预测
可直接生成下一步销售行动建议

SalesFunnel_Frame	Frame	宽 600px，高自适应	容纳漏斗所有阶段和 AI 行动面板	可滚动；顶部固定 Header
Opportunity_Header	Frame	高 80px	商机总览：商机名称、客户、总赢率、阶段进度条	点击折叠/展开商机详情，Hover 显示预测金额
2. 阶段模块（Stage Module）
每阶段统一组件结构，可复制六次，纵向排列：

组件名称	类型	属性	功能说明	交互动作
StageHeader	Frame	高 50px	显示阶段名称、阶段赢率、风险状态标签	点击折叠/展开本阶段 KeyEvents、CustomerActivity、PersonaInsights
StageName	Text	字号 16pt，加粗	阶段名称显示	无
StageWinRate	Text + 图标	数字 + 箭头	显示阶段当前赢率及趋势	自动刷新更新
StageStatusTag	Tag	高/中/低风险颜色	阶段风险状态提示	无
KeyEvents	List	高度可变	标志事件列表	点击“添加事件” → 弹出表单输入事件
EventItem	Frame	高 40px	单条事件显示：时间、类型、影响、备注	点击编辑/删除事件；新增事件刷新 StageWinRate
CustomerActivity	Frame	高 120px	雷达图/条形图显示客户主动性	Hover 显示详细指标；动态调整阶段赢率
PersonaInsights	Frame	高 120px	客户个人画像：职位、兴趣、偏好、态度评分、预测行为	Hover 展示详细信息；点击展开历史互动记录
3. AI 行动面板（Footer）
组件名称	类型	属性	功能说明	交互动作
AI_ActionPanel	Frame	高 150px，固定底部	显示 AI 推荐动作列表	点击动作 → 弹出操作界面；执行后刷新 StageWinRate
ActionItem	Frame	高 40px	单条推荐操作	点击执行 → 更新赢率与主动性评分
4. 可视化与颜色设计
元素	颜色	说明
高赢率	绿色	WinRate 高于阈值（>70%）
中等赢率	黄色	WinRate 50%-70%
低赢率	红色	WinRate <50%
高风险事件	红色高亮	标志事件影响较大
客户主动性高	蓝色	雷达图/条形图显示
5. 交互逻辑
事件输入 → 赢率刷新：KeyEvents 新增/修改事件 → 自动刷新 StageWinRate
阶段折叠/展开：StageHeader 点击 → 展示或隐藏详细组件
客户主动性分析 → 赢率修正：雷达图/条形图动态更新阶段赢率
AI 推荐动作 → 点击执行：执行动作 → 自动更新当前阶段和下一阶段赢率
跨阶段预测：当前阶段事件 + 客户画像 → 预测下一阶段赢率并显示
What-if 模拟：调整事件权重 → 动态更新预测结果
6. 导出/交互增强
支持导出 PDF 或分享链接
事件排序（时间 / 影响值）
阶段筛选（仅显示关键阶段或高风险阶段）
模拟不同策略对预测结果的可视化显示
