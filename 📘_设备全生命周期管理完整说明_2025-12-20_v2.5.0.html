系统概述

1.1 系统定位

设备运营大脑是一个集成了设备全生命周期管理的智能系统，覆盖设备从采购、安装、运行、维护到报废的全过程。

1.2 核心价值

  • 提升设备可用率: 从75%提升至90%+
  • 降低维护成本: 节省30%维修费用
  • 延长设备寿命: 平均延长2-3年
  • 提高生产效率: OEE从65%提升至85%

1.3 技术架构

├─────────────────────────────────────────┤

└─────────────────────────────────────────┘

2.3 设备信息字段

A. 基本信息

  • 设备编号: 唯一标识（如CNC-001）
  • 设备名称: 中文名称
  • 设备分类:
    • 加工设备（CNC、车床、铣床）
    • 装配设备（机器人、拧紧设备）
    • 检测设备（三坐标、硬度计）
    • 辅助设备（空压机、冷却系统）
  • 制造商: 厂家名称
  • 型号: 具体型号
  • 采购日期: YYYY-MM-DD
  • 安装日期: YYYY-MM-DD
  • 启用日期: YYYY-MM-DD

B. 技术参数

  • 额定功率: 单位kW
  • 工作压力: 单位MPa
  • 工作温度: 单位℃
  • 运行速度: 单位rpm/m/min
  • 加工精度: 单位μm
  • 重量: 单位kg
  • 尺寸: 长×宽×高(mm)

C. 财务信息

  • 原值: 采购价格（元）
  • 折旧方式: 直线折旧/加速折旧
  • 折旧年限: 年
  • 残值率: %
  • 当前净值: 自动计算
  • 累计折旧: 自动计算

D. 位置信息

  • 所属车间: 下拉选择
  • 所属产线: 下拉选择
  • 具体位置: 文本描述
  • GPS坐标: 可选（大型设备）

E. 责任人信息

  • 设备主管: 选择员工
  • 操作员: 多选
  • 维护工程师: 选择员工
  • 联系电话: 多个

2.4 设备状态标识

状态灯颜色: - 🟢 运行中 (绿色): 设备正常运行 - 🟡 待机中 (黄色): 设备待命状态 - 🔴 故障中 (红色): 设备发生故障 - ⚫ 停机中 (灰色): 设备计划停机 - 🔵 维护中 (蓝色): 正在进行维护

健康指数颜色: - 90-100分: 🟢 优秀（绿色） - 70-89分: 🔵 良好（蓝色） - 50-69分: 🟡 预警（黄色） - <50分: 🔴 故障（红色）

设备效率分析

5.1 OEE综合效率

OEE计算公式:

OEE = 可用率 × 性能率 × 质量率

可用率 = (计划运行时间 - 停机时间) / 计划运行时间
性能率 = (实际产量 × 标准节拍) / 实际运行时间
质量率 = (合格品数量 / 总产量)

A. 数据库字段说明

设备表 (equipment):

CREATE TABLE equipment (
  id INTEGER PRIMARY KEY,
  code VARCHAR(50) UNIQUE,        -- 设备编号
  name VARCHAR(100),               -- 设备名称
  category VARCHAR(50),            -- 设备分类
  manufacturer VARCHAR(100),       -- 制造商
  model VARCHAR(100),              -- 型号
  purchase_date DATE,              -- 采购日期
  install_date DATE,               -- 安装日期
  start_date DATE,                 -- 启用日期
  status VARCHAR(20),              -- 状态
  health_index DECIMAL(5,2),       -- 健康指数
  running_hours INTEGER,           -- 运行时长
  location VARCHAR(200),           -- 位置
  responsible_person VARCHAR(50),  -- 责任人
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

维修工单表 (maintenance_order):

CREATE TABLE maintenance_order (
  id INTEGER PRIMARY KEY,
  order_no VARCHAR(50) UNIQUE,     -- 工单号
  equipment_id INTEGER,            -- 设备ID
  fault_type VARCHAR(50),          -- 故障类型
  priority VARCHAR(20),            -- 优先级
  description TEXT,                -- 故障描述
  status VARCHAR(20),              -- 状态
  engineer VARCHAR(50),            -- 维修工程师
  start_time TIMESTAMP,            -- 开始时间
  end_time TIMESTAMP,              -- 结束时间
  cost DECIMAL(10,2),              -- 费用
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
