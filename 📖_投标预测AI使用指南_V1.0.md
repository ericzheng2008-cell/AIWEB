# 📖 投标预测AI系统使用指南 V1.0

## 🎯 快速开始（3分钟上手）

### 第1步：进入系统
1. 打开浏览器访问：`http://localhost:5173`
2. 点击顶部导航 **"明升 AICRM"**
3. 点击菜单栏 **"投标预测AI"** 标签

### 第2步：选择项目
- 左侧项目列表点击任意项目
- 系统自动加载项目数据和初始预测

### 第3步：调整报价
- 拖动底部报价滑块
- 观察实时中标概率和期望利润变化
- 查看风险提示（绿色=推荐，橙色=谨慎，红色=警告）

### 第4步：提交报价
- 点击 **"提交第X轮报价"** 按钮
- 系统记录本轮数据并更新预测

## 🎮 核心功能使用

### 1. 三轮投标流程

#### 第1轮：初始预测
1. 选择项目后自动显示初始概率
2. 查看建议报价区间（绿色条形图）
3. 调整报价滑块观察概率变化
4. 点击 **"提交第1轮报价"**

#### 第2轮：动态优化
1. 点击 **"进入下一轮"** 按钮
2. 系统自动更新竞对预测（±10%调整）
3. 重新调整报价
4. 点击 **"提交第2轮报价"**

#### 第3轮：最终优化
1. 再次点击 **"进入下一轮"**
2. 查看三轮累计数据
3. 使用模拟器验证策略
4. 提交最终报价

### 2. 报价滑块操作

#### 基本操作
- **鼠标拖动**：左右拖动滑块调整报价
- **键盘控制**：点击滑块后使用←→方向键微调
- **步进值**：每次调整1000元

#### 实时反馈
```
拖动滑块 → 立即计算 → 更新显示
              ↓
    ┌─────────┴────────┐
    │ 中标概率         │
    │ 期望利润         │
    │ 利润率           │
    │ 风险提示         │
    └──────────────────┘
```

### 3. 风险提示系统

#### 三级警告
| 提示类型 | 触发条件 | 颜色 | 建议 |
|---------|---------|------|------|
| ✅ 报价优秀 | 概率≥60% 且 利润≥15% | 绿色 | 建议提交 |
| ⚠️ 谨慎考虑 | 概率30-60% 或 利润10-15% | 橙色 | 优化报价 |
| ❌ 高风险 | 概率<30% 或 利润<10% | 红色 | 不建议投标 |

#### 示例提示
```
✅ 报价优秀
当前报价平衡了中标概率和利润，建议提交

⚠️ 中标风险高
当前报价中标概率较低，建议降低报价或重新评估

⚠️ 利润不足
当前报价利润率过低，建议适当提高报价
```

### 4. 多轮模拟器

#### 打开模拟器
- 点击左侧 **"打开模拟器"** 按钮
- 或点击顶部 **"运行模拟"** 入口

#### 参数设置
1. **竞对数量**：2-8个（默认4个）
2. **竞对策略**：
   - 保守：高报价策略
   - 中等：平衡策略
   - 激进：低报价策略
   - 混合：随机组合
3. **我方报价策略**：滑块1-100（1=最低，100=最高）

#### 运行模拟
```
点击 [运行模拟] 
    ↓
蒙特卡洛模拟1000次（约2秒）
    ↓
输出结果：
├─ 平均中标概率: 68%
├─ 期望利润: ¥52万
├─ 风险等级: 低
└─ AI推荐: ✅ 推荐提交
```

#### 结果解读
- **柱状图**：显示各轮中标概率变化
- **统计数据**：平均值提供决策参考
- **AI推荐**：
  - ✅ 推荐提交（绿色）：概率≥70%
  - ⚠️ 谨慎考虑（橙色）：概率50-70%
  - ❌ 不建议投标（红色）：概率<50%

## 📊 数据解读

### 中标概率
```
85-95%  非常高  │ ━━━━━━━━━━━━━━━ │ 绿色
70-85%  高      │ ━━━━━━━━━━━     │ 浅绿
50-70%  中等    │ ━━━━━━          │ 橙色
30-50%  较低    │ ━━━             │ 浅红
5-30%   很低    │ ━               │ 红色
```

### 期望利润
```
期望利润 = (报价 - 成本) × (中标概率 / 100)

示例：
报价 ¥440万
成本 ¥350万
概率 75%
期望利润 = (440 - 350) × 0.75 = ¥67.5万
```

### 利润率
```
利润率 = (报价 - 成本) / 报价 × 100%

示例：
报价 ¥440万
成本 ¥350万
利润率 = (440 - 350) / 440 × 100% = 20.5%

评级：
≥25%  优秀  绿色
15-25% 良好  浅绿
10-15% 一般  橙色
<10%   偏低  红色
```

## ❓ 常见问题

### Q1: 中标概率为什么不是100%？
**A**: AI预测基于概率模型，考虑多种不确定因素（竞对策略、客户偏好等），100%概率不现实也不科学。通常60-80%为合理范围。

### Q2: 期望利润为负数怎么办？
**A**: 说明报价低于成本！立即提高报价至建议区间内。系统会用红色警告提示。

### Q3: 模拟器结果和实际不一致？
**A**: 模拟器基于统计模型，提供参考而非预言。实际投标受更多因素影响（政策、关系等）。

### Q4: 如何提高预测准确性？
**A**: 
1. 定期更新客户互动数据
2. 记录历史投标结果
3. 收集真实竞对报价数据
4. 未来版本将接入真实ML模型

### Q5: 三轮报价如何调整？
**A**: 
- 第1轮：建议区间中上（试探）
- 第2轮：根据竞对降低5-10%
- 第3轮：微调至最优点（±2-3%）

### Q6: 风险提示出现矛盾怎么办？
**A**: 
- 概率高但利润低 → 权衡取舍，优先保证利润
- 概率低但利润高 → 考虑放弃或大幅降价

## 🔧 故障排除

### 问题1：页面显示空白
```
解决方案：
1. 刷新浏览器（Ctrl + F5）
2. 检查控制台错误（F12）
3. 确认服务器运行中
```

### 问题2：滑块拖动卡顿
```
解决方案：
1. 关闭其他浏览器标签
2. 清除浏览器缓存
3. 使用Chrome/Edge浏览器
```

### 问题3：模拟器无响应
```
解决方案：
1. 等待2-3秒（1000次模拟需要时间）
2. 减少竞对数量（8→4）
3. 查看浏览器控制台日志
```
