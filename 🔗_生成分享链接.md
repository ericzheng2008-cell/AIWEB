# 🔗 生成可分享的网站链接

## 🎯 三种方案供您选择

---

## 方案一：在线部署（推荐 - 任何人都可访问）

### 🌟 Netlify Drop - 最简单！

**步骤：**

1. **打开浏览器**，访问：
   ```
   https://app.netlify.com/drop
   ```

2. **找到 `dist` 文件夹**
   - 位置：`C:\Users\EricZ\CodeBuddy\AIWEB1\dist`

3. **拖拽 `dist` 文件夹**到网页中

4. **等待上传完成**（约30秒）

5. **获得链接**，格式如：
   ```
   https://amazing-site-123456.netlify.app
   ```

6. **分享链接给同事**即可！

**优点：**
- ✅ 完全免费
- ✅ 无需注册（可选）
- ✅ 全球任何地方都可访问
- ✅ 自动 HTTPS 加密
- ✅ 永久有效（除非手动删除）

---

## 方案二：局域网访问（同事需在同一网络）

### 📡 使用本地预览服务器

**步骤：**

1. **双击运行：** `本地预览并分享.bat`

2. **查看输出的链接**，例如：
   ```
   局域网访问：http://192.168.1.100:4173
   ```

3. **分享此链接**给同一局域网的同事

4. **保持命令窗口开启**（关闭后链接失效）

**优点：**
- ✅ 快速，无需上传
- ✅ 数据不离开本地网络
- ✅ 完全免费

**缺点：**
- ❌ 同事必须在同一局域网
- ❌ 您的电脑必须保持开机
- ❌ 服务必须保持运行

---

## 方案三：使用 Netlify CLI（自动化）

### 🤖 命令行部署

**步骤：**

1. **双击运行：** `一键部署到Netlify.bat`

2. **首次使用**会要求：
   - 安装 Netlify CLI（自动）
   - 登录 Netlify 账号（浏览器打开）
   - 授权访问

3. **等待部署完成**（约1-2分钟）

4. **获得链接**，例如：
   ```
   https://your-site-name.netlify.app
   ```

5. **分享链接**给同事

**优点：**
- ✅ 可以自定义域名
- ✅ 可以更新网站（重新运行脚本）
- ✅ 有管理后台

## 🆘 遇到问题？

### Netlify Drop 上传失败
- 检查网络连接
- 尝试使用代理/VPN
- 确保 `dist` 文件夹完整

### 本地预览无法访问
- 检查防火墙设置
- 确保同事在同一网络
- 确认端口 4173 未被占用

### Netlify CLI 登录失败
- 检查浏览器是否阻止弹窗
- 手动访问授权链接
- 检查网络连接
