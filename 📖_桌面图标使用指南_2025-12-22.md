## 📌 功能详解

### 功能1: 🚀 一键启动 AIWEB

**作用**: 启动前后端服务器，并自动打开浏览器

**执行流程**:
1. 检查端口3000（后端）和3002（前端）
2. 如未运行则自动启动服务
3. 等待8秒初始化
4. 自动打开浏览器访问 http://localhost:3002

**特点**:
- ✅ 智能检测，避免重复启动
- ✅ 自动打开浏览器
- ✅ 显示详细启动状态

**使用场景**: 每次想使用AIWEB时

---

### 功能2: ⚙️ 设置开机自启动

**作用**: 配置Windows开机自动启动AIWEB

**子菜单**:
```
[1] ✅ 启用开机自启动
    → 开机后自动在后台启动
    → 无窗口，静默运行
    
[2] ❌ 禁用开机自启动
    → 取消开机自启动
    
[3] 🔍 查看当前状态
    → 查看是否已启用
    → 显示启动文件位置
```

**启用后效果**:
- 开机自动启动AIWEB
- 后台运行，不显示窗口
- 随时打开浏览器访问 http://localhost:3002

**启动文件位置**:
```
C:\Users\你的用户名\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\AIWEB_Auto_Start.vbs
```

**使用场景**: 每天都要使用AIWEB

---

### 功能3: 🔍 查看服务状态

**作用**: 检查前后端服务是否运行

**显示信息**:
```
✅ 后端服务 (端口3000): 运行中
   进程ID: 12345

✅ 前端服务 (端口3002): 运行中
   进程ID: 67890
```

或
```
❌ 后端服务: 未运行
❌ 前端服务: 未运行
```

**使用场景**: 检查服务是否正常运行

---

### 功能4: 🛑 停止所有服务

**作用**: 强制停止所有AIWEB相关服务

**执行流程**:
1. 显示确认提示
2. 输入 Y 确认
3. 查找并终止端口3000和3002的进程
4. 显示停止结果

**特点**:
- ⚠️ 需要确认操作
- ✅ 安全终止进程
- ✅ 显示被终止的进程ID

**使用场景**: 
- 需要重启服务时
- 释放系统资源时
- 服务出现问题时

---

### 功能5: 📌 创建桌面快捷方式

**作用**: 在桌面创建快捷方式

**执行流程**:
1. 检测桌面路径
2. 使用PowerShell创建.lnk文件
3. 设置图标名称为 `🚀 明升AIWEB`
4. 显示创建结果

**快捷方式属性**:
- 目标: 🚀_明升AIWEB_智能启动.bat
- 工作目录: 项目根目录
- 描述: 明升AIWEB智能启动控制中心

**使用场景**: 
- 快捷方式丢失时
- 重新安装系统后
- 需要在其他位置创建时

## ❓ 常见问题

### Q1: 桌面图标丢失了怎么办？

**A**: 
```
方法1（推荐）: 
双击项目中的 🚀_明升AIWEB_智能启动.bat
选择 [5] 创建桌面快捷方式

方法2:
双击项目中的 ⚡_一键完成所有配置.bat
自动重新创建
```

---

### Q2: 双击桌面图标没反应？

**A**: 
```
1. 右键桌面图标 → 属性
2. 检查"目标"路径是否正确
3. 如果路径错误或文件不存在，重新创建快捷方式

正确路径示例:
C:\Users\EricZ\CodeBuddy\AIWEB1\🚀_明升AIWEB_智能启动.bat
```

---

### Q3: 开机后没有自动启动？

**A**:
```
1. 双击桌面图标
2. 选择 [2] 设置开机自启动
3. 选择 [3] 查看当前状态
4. 如果未启用，选择 [1] 启用开机自启动
5. 重启电脑测试
```

---

### Q4: 如何禁用开机自启动？

**A**:
```
1. 双击桌面图标
2. 选择 [2] 设置开机自启动
3. 选择 [2] 禁用开机自启动
4. 完成！
```

---

### Q5: 服务启动失败怎么办？

**A**:
```
1. 双击桌面图标
2. 选择 [4] 停止所有服务（如果有服务在运行）
3. 等待10秒
4. 选择 [1] 一键启动
5. 查看命令窗口是否有错误提示
```

---

### Q6: 能同时打开多个AIWEB吗？

**A**: 
不能！服务已经做了端口检测：
- 如果服务已在运行，再次启动会跳过
- 一台电脑同一时间只能运行一个AIWEB实例

---

### Q7: 如何卸载/删除？

**A**:
```
1. 双击桌面图标
2. 选择 [2] 设置开机自启动
3. 选择 [2] 禁用开机自启动
4. 删除桌面快捷方式
5. 删除项目文件夹
```

## 🔧 故障排查

### 问题: 启动后浏览器显示"无法访问"

**原因**: 服务未完全启动或端口被占用

**解决**:
```
1. 等待15秒（服务需要初始化时间）
2. 刷新浏览器
3. 如果还不行，重新启动服务
```

---

### 问题: 命令窗口闪退

**原因**: 批处理文件路径有问题或Node.js未安装

**解决**:
```
1. 检查Node.js是否安装（打开CMD，输入 node -v）
2. 确认项目文件夹路径正确
3. 右键快捷方式 → 属性 → 检查"目标"和"起始位置"
```

---

### 问题: 端口被占用

**症状**: 启动时提示端口3000或3002被占用

**解决**:
```
方法1: 停止占用端口的程序
1. 双击桌面图标
2. 选择 [4] 停止所有服务

方法2: 手动查找并终止
1. 打开任务管理器
2. 找到 node.exe 进程
3. 结束任务
```
