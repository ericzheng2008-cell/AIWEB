## 上传图片

### 方法1: 本地上传（推荐）

```
在任何需要图片的字段中，点击上传区域
```

#### 步骤2: 选择上传方式
```

  - 将图片文件拖到上传区域
  - 自动上传

方式B: 点击选择
  - 点击上传区域
  - 在弹出的对话框中选择文件
```

#### 步骤3: 等待上传完成
```

✅ JPG / JPEG
```

```
📏 尺寸: 1200x600 (Banner) / 400x400 (图标) / 600x400 (配图)
```

### 数据持久化

#### 当前存储方式
```
📦 使用浏览器 localStorage 存储
📦 数据保存在本地浏览器中
📦 刷新页面后数据仍然存在
```

#### 注意事项
```
⚠️ 清空浏览器数据会丢失内容
⚠️ 不同浏览器数据不共享
⚠️ 建议定期备份重要内容
```

#### 验证保存
```
1. 保存内容后
2. 刷新页面（Ctrl+F5）
3. 再次进入"内容编辑"标签页
4. 确认内容仍然存在
```

## ❓ 常见问题

### Q1: 保存后刷新页面，内容丢失了？
```
A: 检查以下几点:
1. 确认点击了"保存所有内容"按钮
2. 看到"保存成功"提示信息
3. 没有清空浏览器缓存/数据
4. 使用相同的浏览器访问
```

### Q2: 图片上传失败？
```
A: 可能原因:
1. 图片格式不支持（只支持 JPG/PNG/GIF）
2. 图片太大（建议 <2MB）
3. 网络连接问题
解决方法: 压缩图片后重新上传，或使用URL方式
```

### Q3: 如何恢复默认内容？
```
A: 
1. 打开浏览器开发者工具（F12）
2. 进入 Application → Local Storage
3. 删除 'tightening_page_content' 键
4. 刷新页面
5. 系统会加载默认内容
```

### Q4: 可以导出/备份内容吗？
```
A: 当前版本使用 localStorage 存储
手动备份方法:
1. F12 打开开发者工具
2. Console 输入:
   JSON.stringify(localStorage.getItem('tightening_page_content'))
3. 复制输出结果保存到文件
恢复方法:
1. F12 打开开发者工具
2. Console 输入:
   localStorage.setItem('tightening_page_content', '粘贴备份内容')
```

### Q5: 图片显示不出来？
```
A: 检查:
1. 图片URL是否正确
2. 图片链接是否有效（未失效）
3. 图片是否允许跨域访问
4. 浏览器是否阻止HTTP图片（使用HTTPS）
```
